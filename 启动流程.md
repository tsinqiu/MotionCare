# MotionCare 启动流程

项目实际根目录：

```powershell
cd C:\school\course\3.2\MotionCare\MotionCare
```

## 1. 确认 MySQL 已启动

```powershell
Get-Service MySQL
```

如果不是 `Running`：

```powershell
Start-Service MySQL
```

后端默认读取：

```text
backend/.env
```

当前数据库配置使用 `MotionAnalysis`，账号密码见本地 `.env`。

## 2. 启动后端

```powershell
cd C:\school\course\3.2\MotionCare\MotionCare\backend
npm start
```

后端地址：

```text
http://127.0.0.1:8080/api
```

健康检查：

```powershell
Invoke-RestMethod http://127.0.0.1:8080/api/health
```

正常时应看到：

```text
status: ok
database.message: connected
```

## 3. 启动前端

新开一个 PowerShell：

```powershell
cd C:\school\course\3.2\MotionCare\MotionCare\frontend
npm run dev -- --host 0.0.0.0
```

前端地址：

```text
http://localhost:5173/
```

## 4. 端口被占用时重启

检查监听进程：

```powershell
Get-NetTCPConnection -LocalPort 5173,8080 -ErrorAction SilentlyContinue |
  Where-Object { $_.State -eq 'Listen' } |
  Select-Object LocalAddress,LocalPort,State,OwningProcess
```

停止占用端口的进程：

```powershell
Stop-Process -Id <OwningProcess> -Force
```

然后重新执行后端和前端启动命令。

## 5. 后台启动方式

如果不想占用两个终端，可以在项目根目录执行：

```powershell
$root = "C:\school\course\3.2\MotionCare\MotionCare"

Start-Process -FilePath "npm.cmd" `
  -ArgumentList @("start") `
  -WorkingDirectory "$root\backend" `
  -RedirectStandardOutput "$root\backend.log" `
  -RedirectStandardError "$root\backend.err" `
  -WindowStyle Hidden

Start-Process -FilePath "npm.cmd" `
  -ArgumentList @("run", "dev", "--", "--host", "0.0.0.0") `
  -WorkingDirectory "$root\frontend" `
  -RedirectStandardOutput "$root\frontend-dev.out.log" `
  -RedirectStandardError "$root\frontend-dev.err.log" `
  -WindowStyle Hidden
```

检查结果：

```powershell
Invoke-RestMethod http://127.0.0.1:8080/api/health
Invoke-WebRequest http://localhost:5173/
```
